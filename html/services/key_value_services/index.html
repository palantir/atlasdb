Key Value Services¶

Overview¶

This matrix details the current state of available Key Value Services supported by the AtlasDB development staff.

Key Value Service State KVS Versions Supported Recommended Versions Errata
Cassandra KVS Supported 2.2.9+ 2.2.9  
DB KVS (Postgres) Supported 9.2+ 9.3.4 Amazon RDS explicitly supported, other HA postgres variants not currently supported
DB KVS (Oracle) Beta Support 11g+ 12c Amazon RDS explicitly supported, RAC not currently supported
CQL KVS Unsupported      
JDBC KVS Unsupported      

Details¶

  • Cassandra KVS
  • DB KVS
  • CQL KVS
  • JDBC KVS

Cassandra KVS¶

The Cassandra KVS allows AtlasDB to use Apache Cassandra as it’s backing key value service. For it to be sole backing KVS for AtlasDB needs Cassandra Lightweight Transactions, otherwise earlier version of Cassandra can be used to back AtlasDB.

DB KVS¶

DB KVS allows AtlasDB to use Postgres and Oracle as a backing store.

CQL KVS¶

CQL KVS is not currently supported, but we plan on supporting this KVS eventually.

JDBC KVS¶

JDBC KVS existed prior to DB KVS and was the primary means for deploying applications on Postgres. JDBC KVS has been deprecated in favor of DB KVS.

Deleted¶

ClusterKeyValueService.java - Intended to be run on top of a MySQL/Cluster RocksDbKVS - Removed in 0.44
